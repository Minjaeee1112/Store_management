# 매장 관리 시스템 (Store Management System)
웹 기반으로 매장의 주문 · 재고 · 매출 · 테이블 관리를 통합하여 운영 효율을 높이는 시스템입니다.
스마트폰 QR 주문 기능을 제공하여 사용자 편의성을 높이고, 관리자는 웹 관리자 페이지를 통해 매장을 효율적으로 운영할 수 있습니다.

•팀원 소개

-한성재
-김성섭
-조민재

• 프로젝트 개요

-JSP + MySQL 기반의 웹 관리 시스템 구축

-고객은 스마트폰으로 QR을 스캔하여 쉽게 주문

-관리자는 재고, 매출, 주문 내역 등을 한 곳에서 관리 가능


•프로젝트 개발 배경

기존 매장에서 겪는 문제점들을 해결하기 위해 개발되었습니다.


!!문제점!!

-수동적인 재고 관리로 인한 비효율

-키오스크 및 메뉴판 이용의 불편함

-객관적인 매장 분석 자료 부족


•기대 효과

-쉽고 빠른 주문 흐름 제공

-고객 만족도 향상

-효과적인 재고 및 매출 관리


•프로젝트 목적

- 매장의 재무·재고 상태를 자동화하여 효율적 관리

-관리자가 간편하게 운영할 수 있는 직관적인 시스템 제공

-고객이 QR 기반으로 편하게 주문할 수 있는 UX 구현


• 유스케이스 다이어그램

사용자(User)

-메뉴 조회

-장바구니 담기/삭제

-주문 · 결제

-주문 내역 조회

관리자(Admin)

-상품 등록/수정/삭제

-품절 관리

-주문 관리

-결제 및 취소 관리

-매출 관리



•WBS (Work Breakdown Structure)


1. 메뉴 관리

-상품 등록

-상품 수정

-상품 삭제

-품절 관리


2. 결제 관리

-결제

-취소


3. 로그인 관리

-로그인/로그아웃

-아이디 찾기

-비밀번호 찾기


4. 매출 관리

-손익 계산서


5. 주문 관리

-메뉴조회

-장바구니

-주문내역 관리

-테이블 관리


6. 디버깅 & 수정


- ERD (Entity Relationship Diagram)

시스템 주요 테이블 구조:

-User / UserSession

-Product

-Order / OrderItem / OrderTable

-Payment
